LAYOUT & POSITIONING

About

When all the elements and their box models are defined, the next challenge is telling the web browser how you want them to be displayed. By default, all the element box model rectangles will be displayed one under another and their position will be set to static and to change this, we have to modify the position property. The valid values for this property are: static, relative, fixed and absolute Each one effects how the box is positioned when the top, bottom, left and right properties are modified. The position must be set first however, otherwise they will not work.
If position: static;, the top, bottom, left and right properties will have no effect on the element. It will be simply follow the natural flow of the website.

If position: relative;, like static, the content will follow the natural flow of the page however it's position will be modifiable with top, bottom, left and right properties. After movement, other elements will not try and fill the gap that was left by this element. position: fixed;, will make the element stay in the same place on the screen, even when the page is scrolled. It doenst leave a gap in the natural flow of the page, meaning that it is filled by other elements. Lastly, position: absolute;, is positioned in relation to it's nearest non-static ancestor if one exists or the body if not, in which case, it is free to scroll with the page.

Another helpful property for positioning the elements is float. It is a potentially very neat way to position content and in it's simplest use, could be used to wrap text around images.
